- Board Meetings now requires a real Supabase session instead of treating localStorage user IDs as authorization.

Post-deploy test:

2. In console: await supabaseClient.auth.getSession() should return a session with user.id.
3. Run BOARD_MEMBERS_PRODUCTION_RLS.sql in Supabase SQL Editor.
4. Board Meetings should load assigned communities and active BoardMembers under secure RLS.
